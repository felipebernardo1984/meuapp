1. Estrutura do App
O app precisaria ter três perfis principais de usuário:
1.	Aluno
o	Faz check-in direto pelo app na arena, sem catraca.
o	Vê progresso do plano de 8 ou 12 check-ins (1 ou 2 aulas por semana).
o	Consulta status da mensalidade e dias restantes no ciclo.
2.	Professor
o	Recebe notificação quando aluno faz check-in.
o	Vê lista dos alunos de sua modalidade, planos, últimos check-ins e progresso.
o	Atualiza manualmente check-ins caso necessário.

3.	Gestor / Admin
o	Acesso a todas as modalidades e professores.
o	Gera relatórios consolidados (PDF/Excel) com todos os dados: alunos, planos, check-ins, status de mensalidade, última presença.
o	Configura planos, ciclos e modalidades.
________________________________________
2. Funcionalidades Principais
Para o Aluno
•	Login via CPF/email + senha (opção futura de integração com Wellhub/TotalPass, via OAuth se a API permitir).
•	Botão de Check-in.
•	Barra de progresso: quantidade de check-ins realizados no ciclo.
•	Visualização de plano ativo, status da mensalidade e dias restantes do ciclo.
Para o Professor
•	Lista de alunos de sua modalidade:
o	Nome completo
o	Plano (8x ou 12x)
o	Check-in ✅/❌
o	Último check-in (data/hora) e progresso do chekin
•	Notificação automática quando aluno realiza check-in.
•	Botão para atualizar lista ou corrigir check-ins ou concluir o ciclo ( deixar no app todos os ciclos para conferencia futura

Para o Gestor
•	Filtros por modalidade, professor, data.
•	Relatórios detalhados:
o	Nome do aluno
o	Modalidade
o	Plano (8/12)
o	Check-ins realizados
o	Status da mensalidade
o	Último check-in
o	Ciclos concluídos ou alunos pendentes
•	Exportação PDF / Excel.
•	Gerenciamento de contas de professores e alunos.
________________________________________
3. Regras do Plano de Fidelidade
•	Ciclo de 30 dias:
o	Plano 8x → 1x/semana
o	Plano 12x → 2x/semana
•	Contagem de check-ins reinicia ao final do ciclo ou ao término do plano, mas fica icone de ciclo concluído ou aluno pendente.
•	Permite ajuste manual pelo professor (em casos de falhas ou erros).
________________________________________
4. Integração com Wellhub / TotalPass
•	Ideal: se ambas plataformas possuem API, você pode:
o	Usar login único (SSO): aluno entra com credenciais do Wellhub ou TotalPass.
o	Ler check-ins automaticamente e replicar no app Arena MUV.
•	Caso não tenha API: o aluno faria check-in manualmente no app Arena MUV, enquanto o professor confirma o controle.
________________________________________
5. Tecnologias Recomendadas
•	Frontend (App): Flutter ou React Native → funciona em Android e iOS com uma única base de código.
•	Backend / Banco de Dados:
o	Node.js + Express / Django / Laravel
o	Banco: PostgreSQL ou Firebase (mais rápido para MVP)
•	Notificações: Firebase Cloud Messaging
•	Relatórios PDF/Excel: bibliotecas no backend (ex: jsPDF, Pandas/Excel).
•	Autenticação: JWT para API, OAuth para integração futura com Wellhub/TotalPass.
________________________________________
6. Fluxo de Uso (Protótipo Funcional)
1.	Aluno
o	Abre app → faz login → clica em “FAZER CHECK-IN” → barra de progresso atualizada → notificação enviada ao professor.
2.	Professor
o	Abre app → visualiza lista de alunos → vê check-ins atualizados → recebe notificação automática → corrige manualmente se necessário.
3.	Gestor/Admin
o	Abre app → acessa relatório → filtra por modalidade/professor/datas → exporta PDF ou Excel.
